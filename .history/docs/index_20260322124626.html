3. Dynamic fields schema

La riga id è sempre presente come campo di sistema. Tutti gli altri campi si aggiungono una alla volta tramite il builder.

name ? type ? required ? computed ? visibleInForm ? visibleInViewer ? enumValues ? actions ?

Regole pratiche del builder:
1) id resta sempre presente come campo di sistema.
2) Se un campo è computed = YES, allora required = NO e visibleInForm = NO.
3) Se il tipo non è enum, l'area enumValues viene ignorata.
4) Per enumValues si inserisce un valore per riga.
