Generate your app

Carica i file esportati da Google Sheets per avviare il flusso di generazione. Il sistema userà questi file per leggere la struttura del database e preparare il frontend CRUD, il viewer e il backend Apps Script.

1. Upload source files

I file richiesti sono l’export .xlsx del foglio e l’export .html usato per leggere stili, etichette e colori.

2. Parser result

In questa fase il sistema rileva campi, formule, valori enum e impostazioni visuali. Nel flusso completo, questa parte alimenta internamente la configurazione del progetto.

  • Campi rilevati dal foglio
  • Valori enum e relativi colori
  • Formule presenti nei campi calcolati
  • Configurazione di base del frontend
Open internal review page

3. Generated files

Una volta completata la pipeline, i tre file finali del progetto saranno:

  • index.html — home page del database e CRUD finale
  • viewer.html — viewer tabellare
  • codice.gs — backend Google Apps Script
