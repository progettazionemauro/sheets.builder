Carica i file esportati da Google Sheets e prepara la generazione del progetto finale. In questa fase il parser e il generatore reali continuano a essere gestiti dalla pipeline Python, ma questa pagina rappresenta il flusso cliente corretto.

1. Source files

I tre input logici di partenza sono: .xlsx, .html e sheet name. Il nome del foglio è obbligatorio perché viene usato nella configurazione del progetto.

2. Parser status

Il parser legge la struttura del database, i campi, le formule rilevate, gli enum e i colori. In questa fase intermedia, il controllo tecnico dettagliato può ancora essere rifinito nella pipeline interna.

  • Campi rilevati dal foglio
  • Formule trovate nei campi calcolati
  • Enum e colori visuali
  • Configurazione di base del frontend

3. Generate final files

Dopo il parser, la pipeline genera i tre file finali del progetto: index.html, viewer.html e codice.gs.

Final package prepared

  • index.html — home page del database e CRUD finale
  • viewer.html — viewer tabellare
  • codice.gs — backend Google Apps Script
